Thuật toán Tarjan

Tìm tất cả Thành phần Liên thông Mạnh (Strongly Connected Components) trong đồ thị có hướng — thời gian O(V + E).

Khái niệm cơ bản

Trước khi bắt đầu

🔗
SCC là gì?
Thành phần liên thông mạnh — tập hợp đỉnh mà mọi cặp đỉnh đều có thể đi đến nhau.
🕰️
disc[u]
Thời điểm khám phá đỉnh u khi chạy DFS. Mỗi đỉnh có giá trị duy nhất, tăng dần.
🔻
low[u]
Giá trị disc nhỏ nhất có thể đạt được từ cây con DFS của u, kể cả qua back-edge.
📦
Stack
Lưu trữ các đỉnh đang được xét. Khi low[u] == disc[u], pop stack để lấy 1 SCC.
Ví dụ minh họa

Đồ thị mẫu & 3 SCC

Kết quả: Đồ thị trên có 3 SCC: {0,1,2} · {3,4} · {5,6,7,8}. Các cạnh đứt giữa SCC là cạnh liên SCC không tạo thành chu trình.
Cách hoạt động

Từng bước thuật toán

01

Khởi tạo biến toàn cục

Tạo mảng disc[], low[], inStack[], stack STL, và bộ đếm thời gian timer. Tất cả đặt về -1 / false / 0.

02

Chạy DFS từ mỗi đỉnh chưa thăm

Với mỗi đỉnh u chưa được thăm (disc[u] == -1), gọi hàm dfs(u). Điều này đảm bảo xử lý toàn bộ đồ thị dù không liên thông.

03

Gán disc[u] và low[u], push vào stack

Khi lần đầu thăm đỉnh u, đặt disc[u] = low[u] = timer++ rồi đẩy u vào stack và đánh dấu inStack[u] = true.

04

Duyệt các đỉnh kề

Với mỗi cạnh u → v: nếu v chưa thăm, đệ quy DFS rồi cập nhật low[u] = min(low[u], low[v]). Nếu v đang trong stack, cập nhật low[u] = min(low[u], disc[v]).

05

Kiểm tra điều kiện tạo SCC

Sau khi xử lý xong mọi cạnh kề của u, nếu low[u] == disc[u] thì u là gốc của một SCC. Pop stack cho đến khi lấy ra u — tập các đỉnh đó tạo thành 1 SCC.

Triển khai

Code C++ hoàn chỉnh

C++ · tarjan.cpp 
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 1e5 + 5;

int n, m;
vector<int> adj[MAXN];
int  disc[MAXN], low[MAXN];
bool inStack[MAXN];
stack<int> st;
int  timer_ = 0;
vector<vector<int>> sccs;    // lưu kết quả các SCC

void dfs(int u) {
    // Gán thời điểm khám phá và giá trị low ban đầu
    disc[u] = low[u] = timer_++;
    st.push(u);
    inStack[u] = true;

    for (int v : adj[u]) {
        if (disc[v] == -1) {
            // v chưa thăm → DFS tiếp
            dfs(v);
            low[u] = min(low[u], low[v]);
        } else if (inStack[v]) {
            // v đang trong stack → back-edge
            low[u] = min(low[u], disc[v]);
        }
    }

    // Kiểm tra u có phải gốc của SCC không
    if (low[u] == disc[u]) {
        vector<int> scc;
        while (true) {
            int node = st.top(); st.pop();
            inStack[node] = false;
            scc.push_back(node);
            if (node == u) break;
        }
        sccs.push_back(scc);
    }
}

void tarjan() {
    memset(disc, -1, sizeof(disc));
    memset(inStack, false, sizeof(inStack));
    for (int i = 0; i < n; i++)
        if (disc[i] == -1)
            dfs(i);
}

int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    cin >> n >> m;
    for (int i = 0; i < m; i++) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
    }

    tarjan();

    cout << "So SCC: " << sccs.size() << "\n";
    for (auto& scc : sccs) {
        for (int x : scc) cout << x << " ";
        cout << "\n";
    }
    return 0;
}

Lưu ý

Các điểm cần nhớ

Stack overflow: Với đồ thị rất sâu (V > 10⁵), đệ quy DFS có thể bị stack overflow. Giải pháp: dùng ulimit -s unlimited hoặc chuyển sang DFS iterative.
inStack vs visited: Chỉ cập nhật low[u] với đỉnh đang trong stack, không phải mọi đỉnh đã thăm. Đây là điểm dễ nhầm nhất.
Thứ tự SCC: Tarjan trả về SCC theo thứ tự reverse topological order của DAG thu gọn — rất tiện cho bài toán xử lý SCC theo topo.
